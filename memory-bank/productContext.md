# Product Context

## 存在理由
大量のZipファイル形式の書籍を効率的に管理・閲覧するため。ファイルマネージャーでは困難なメタデータ管理、検索、重複検出、マルチデバイスアクセスを実現する。

## ユーザーフロー

1. **インポート**: Zipを`book_send/`に配置 → Workerが自動検出 → 解析・ハッシュ計算・サムネイル生成 → DB登録
2. **閲覧**: ログイン → 検索/フィルタ → 書籍選択 → ブラウザで閲覧（履歴・評価自動記録）
3. **管理**: メタデータ編集、重複チェック、エクスポート、ライブラリ共有

## データフロー
```
book_send/ → [Worker] → book_library/ (永続保存)
                      → book_cache/   (サムネイル)
                      → PostgreSQL    (メタデータ)
book_fail/ ← エラー時
```

## 非機能要件
- 書籍一覧・検索: 1秒以内、サムネイル: 2秒以内（キャッシュ済み）
- 数万冊管理対応、同時接続: 数十名程度
- パスワードbcryptハッシュ化、JWTステートレス認証
